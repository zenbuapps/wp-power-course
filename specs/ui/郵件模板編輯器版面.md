# 郵件模板編輯器版面（沉浸式）

> Issue #258 — 郵件編輯器編輯區被左右遮蔽
> 對應 Feature：`specs/issue-258-email-editor-layout/郵件編輯器沉浸式版面.feature`
> 對應 Activity：`specs/issue-258-email-editor-layout/郵件編輯器沉浸式版面.mmd`

## 描述

管理員於後台編輯自動化郵件模板時使用的拖拉式（所見即所得）信件編輯器。底層為 `j7-easy-email-extensions` 的 `StandardLayout` 三欄佈局，原為整頁全螢幕設計，嵌入 WordPress 後台後會被 WP 主選單 + Refine 側邊欄擠壓。本規格定義沉浸式版面優化後的呈現與互動行為，屬**純前端版面呈現優化**，不改變 API、資料模型與已儲存信件內容。

實作進入點：`js/src/pages/admin/Emails/Edit/EmailEditor/index.tsx`（`CustomEmailEditor` → `StandardLayout`）、`js/src/pages/admin/Emails/Edit/index.tsx`。

## 三欄結構

| 區域 | 內容 |
|------|------|
| 左側 | 可拖入信件的元件區（文字、圖片、按鈕、分隔線等） |
| 中間 | 信件本體的編輯與預覽畫布（所見即所得） |
| 右側 | 被選取元件的屬性設定區（字體、顏色、間距等） |

## 版面模式

### 沉浸全螢幕模式（預設）

- 進入編輯頁**自動進入**沉浸模式（除非偏好記憶為一般版面）。
- 隱藏 WordPress 後台主選單（`#adminmenu`）與 Refine 側邊欄，三欄編輯器**獨佔瀏覽器可視寬度**。
- 提供「退出全螢幕」按鈕，可隨時返回一般後台版面。

### 一般後台版面

- 保留 WordPress 後台既有選單、頂部工具列、頁尾儲存列。
- 提供「進入全螢幕編輯」按鈕，可再次進入沉浸模式。

### 偏好記憶

- 記住管理員上次選擇的版面模式（沉浸 / 一般），下次進入編輯頁沿用。
- 採 **client-side localStorage** 持久化，**不落後端 DB、無 REST 端點**。

## 版面規則

| 規則 | 值 / 行為 |
|------|-----------|
| 中間畫布最小可視寬度 | **640px**（信件標準 600px + 左右內距）；不足時觸發面板收合 / 橫向捲動 |
| 左 / 右面板收合 | 可各自收合，收合後空間讓給中間畫布，可再次展開叫回 |
| 窄螢幕面板行為 | 三欄展開會使畫布不足 640px 時，左右面板改為**浮動抽屜**——平常收起不佔畫布寬度，點按鈕滑出覆蓋於畫布上方，再點收起 |
| 不重疊原則 | 左側元件區、中間畫布、右側屬性區三者互不重疊遮蔽，皆可正常操作 |

## 驗收斷點（四種螢幕尺寸）

| 斷點 | 視窗寬度 | 期望 |
|------|----------|------|
| 13 吋筆電 | ~1280px | 中間畫布 ≥ 640px（必要時左右面板轉浮動抽屜） |
| 14/15 吋 | ~1440px | 三欄並排，畫布完整顯示信件全貌 |
| 桌機 | ~1920px | 三欄並排，畫布寬鬆 |
| 超寬螢幕 | ~2560px | 三欄並排，畫布寬鬆 |

## 約束

- 實作**優先低風險 CSS / 容器層覆寫**（不動 `j7-easy-email-extensions` 套件本身），必要時再自訂佈局。
- 退出沉浸模式後，不得破壞 WordPress 後台既有選單、工具列、儲存按鈕的顯示與操作。
- 僅改變編輯畫面呈現，**不改變任何已儲存信件內容與寄出排版**（`short_description` JSON / `description` HTML 產出邏輯維持不變）。
