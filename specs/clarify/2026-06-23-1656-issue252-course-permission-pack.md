## Idea（Issue #252 原文摘要，逐字保留關鍵段落）

### 標題：批量觀看的課程權限（課程權限包）

新增一層更上層的概念——「課程權限包」：站長先定義一個權限範圍（全站／某分類標籤／特定課程），
再把這個權限包掛到商品上。買了商品 = 取得整個權限包。

**前置假設（Issue 已標記待 PM 確認）：**
1. 新概念定位：目前商品只能逐一勾選綁定特定課程清單（bind_courses_data）。新增「課程權限包」上層概念。
2. 動態範圍：全站權限與分類/標籤權限為「動態生效」——之後新上架且落在範圍內的課程，持有權限包的用戶自動可觀看，不需站長補開通。
3. 與訂閱整合：權限包掛到訂閱商品時，權限存續跟隨訂閱生命週期（沿用現有「跟隨訂閱」機制：僅 `active` 與 `pending-cancel` 視為有效）。
4. ⚠️ 待確認：權限包是否需要可設定到期日/觀看期限（如「購買後 30 天」），或一律無限期（訂閱情境除外）？Issue 先假設「一次性商品＝無限期、訂閱商品＝跟隨訂閱」。

**三種範圍：**
- 全站課程權限：涵蓋平台上所有課程（含日後新增）。
- 指定分類/標籤課程權限：以課程分類或標籤篩選出範圍（含日後落入此分類/標籤的新課程）。
- 特定課程權限：手動勾選一份固定的課程清單（不隨新增課程擴張）。

**任務目標：**
- 站長後台新增「課程權限包」並命名。
- 站長為權限包選三種範圍之一。
- 站長把權限包連接到一個（或多個）商品（一次性 + 訂閱）。
- 用戶購買並結帳完成後，自動取得權限包，可觀看範圍內所有課程。
- 訂閱商品：權限跟隨訂閱狀態（active/pending-cancel 可看；expired/cancelled 自動失去）。
- 站長後台可檢視、編輯、停用、刪除既有權限包。

**異常流程：**
- 旅程6：未登入訪客購買含課程權限商品 → 要求先登入；訂單未達開通條件 → 尚未取得權限。
- 旅程7：權限範圍外的課程 → 顯示未開通／需購買，引導前往對應銷售方案。

**風險（Issue 已列）：**
- 動態範圍的用戶預期管理（高價新課誤入吃到飽）。
- 權限疊加與優先順序（多來源權限，任一允許即可看；訂閱失效不可誤砍單獨購買的課）。
- 訂閱失效的即時性與提示。
- 範圍變更的回溯影響（縮小範圍/刪除包，影響已購用戶）。
- 與既有「逐課綁定」並存的行為。

### 關鍵架構假設（ASM）

- **ASM-1（核心）**：現行權限模型是「購買時把課程 id 寫進 user meta `avl_course_ids`（static list）」，
  觀看判定 `is_avl()` = `in_array(course_id, avl_course_ids)`。**動態範圍（全站/分類）無法用 static list 表達**，
  因為購買時無法預知日後新增的課程。必須改為「規則式/讀取時計算（compute-on-read）」模型：
  在 `is_avl()` 之上或之內，新增「該用戶是否持有涵蓋此課程的有效權限包」的判定。
  此為本需求最大的技術決策點，需使用者確認方向（見 Q1）。
- **ASM-2**：權限包本身建議為新的 CPT（如 `pc_access_pass`）或 wp_options 設定集合，
  掛到商品用 product meta（類比現行 `bind_courses_data`）。具體儲存載體需定（見 Q7）。

## Q&A（最終決策表，已拍板）

- Q1 觀看判定方向 → A: **compute-on-read**（不 materialize；is_avl 之上加「有效權限包是否涵蓋此課程」即時計算；與 avl_course_ids OR 疊加）
- Q2 分類範圍語意 → A: **多選 term 聯集 OR + 含子分類**（父分類自動涵蓋子分類課程）
- Q3 期限模式 → A: **三模式**：permanent（永久）/ follow_subscription（跟隨訂閱）/ limited（限時 N 天，limit_value+limit_unit）
- Q4 商品掛包關係 → A: **1 商品掛 1 包**（product meta access_pass_id，1:1）
- Q5 與逐課綁定關係 → A: **並集**（兩者並存、效果 OR，不互斥）
- Q6 停用 vs 刪除 → A: **停用保留已購權限/不可掛新商品；刪除才收回 + confirm 二次確認 + 影響 N 位用戶提示**
- Q7 儲存載體 → A: **新 CPT pc_access_pass**（範圍 + 期限 + 狀態，可命名列管）
- Q8 範圍變更 → A: **即時生效（compute-on-read 自然結果）+ UI 警告影響 N 位用戶**
- Q9 範圍外引導 → A: **導去該課自己的銷售頁**（現成，不新增推薦方案設定）
- Q10 後台管理頁 → A: **獨立頁 js/src/pages/admin/AccessPasses/**（React 18 + Refine + Ant Design）
