# Clarify Session 2026-05-21 10:44

## Idea

Issue #226：**課程學員 TAB UI 調整 — 顯示全部 switch 移到欄位標題**

頁面：`/wp-admin/admin.php?page=power-course#/courses/edit/{course_id}` 的「課程學員」TAB。

現況：
- `StudentTable`（`js/src/pages/admin/Courses/Edit/tabs/CourseStudents/StudentTable/index.tsx`）使用 `useColumns` hook 渲染欄位
- `Granted courses` 欄位由 `AvlCoursesList` 元件渲染，**每個 user row 各自有一個** 「顯示全部」switch（小尺寸）
- 該 switch 控制當前 row 「只看本課程 / 顯示該用戶所有授權課程」
- 預設關（只看本課程），由 `useState(!currentCourseId)` 初始化

目標：
- 把 switch **從 row 內移到 th（欄位標題列）** 上，藍色框框標記的「Granted courses」位置
- switch 切換為「**一次套用到全部 user**」的批次開關
- 切換 ON → 表格內所有 user row 同步展開「顯示全部已授權課程」；切換 OFF → 全部 row 收回「只看本課程」

## 背景檔案

- `js/src/pages/admin/Courses/Edit/tabs/CourseStudents/StudentTable/index.tsx` — Course Edit 頁的學員表格容器
- `js/src/components/user/UserTable/hooks/useColumns.tsx` — 共用欄位定義，產出 `Granted courses` column
- `js/src/components/user/AvlCoursesList/index.tsx` — 渲染單一 user 的 avl_courses，現含 switch
- 關聯規格：
  - `specs/features/student/查詢學員列表.feature`
  - `specs/ui/課程學員管理-用戶挑選彈窗.md`（#190 既有 UI 規格，本次調整不變更上半部）

> **歧義偵測結果**：用戶提及的「顯示全部」實際對應到既有的 `showToggle` switch（render 在 `AvlCoursesList`）。已在 issue 第一輪確認用戶要的是「批次切換」而非「個別預設值」。

## Q&A

### Q1 個別 row 的 switch 處理方式

- 題目：把 switch 移到 th 之後，每個 row 內原本的個別 switch 要怎麼處理？
- 選項：
  - A：**完全移除** row 內的 switch，只留 th 一個統一控制
  - B：保留 row 內的 switch，th 的 switch 只是「批次設定預設值」（個別 row 仍可獨立覆寫）
  - C：不確定，請推薦
- **最終決策：A**
- 理由：Issue 標題寫「切換為全部 user 展開」，語意上是一個全域開關；保留兩處 switch 容易讓使用者混淆狀態來源，移除個別 row switch 更直覺。

### Q2 切換到「展開」狀態的預設值

- 題目：頁面剛載入時，th 內的 switch 預設是「開」還是「關」？
- 選項：
  - A：**預設關（只看本課程）** — 與現況一致，避免一進來看到一堆與本課程無關的資料
  - B：預設開（展開全部 user 的全部課程）— 直接呈現完整資訊
  - C：記住上次的選擇（localStorage）
- **最終決策：A**
- 理由：維持現況，避免行為突變；多數情境下管理員只關心本課程的學員進度。

### Q3 switch 狀態的記憶範圍

- 題目：使用者切換 switch 後，狀態保留多久？
- 選項：
  - A：**僅本次頁面停留**（切換 tab、跳到別頁就重置）
  - B：本課程內記憶（用 localStorage 綁定 course_id）
  - C：全站記憶（用 localStorage 全域記住）
- **最終決策：A**
- 理由：最單純、無 side effect。狀態管理用 `useState` 即可，不需 atom / localStorage。

### Q4 th 內 switch 的 UI 呈現

- 題目：「Granted courses」欄位標題與 switch 如何排版？
- 選項：
  - A：**`Granted courses [Show all 🔘]` —— 欄位標題 + switch 同列**（switch 右側帶「Show all」文字 + Tooltip）
  - B：只在 th 放 switch，把「Granted courses」標題改成 switch 標籤
  - C：switch 放欄位標題下方第二行
- **最終決策：A**
- 理由：保留欄位標題語意（i18n key `Granted courses` 沿用），switch 在同行靠右，沿用既有 row 內 switch 的視覺樣式（small size + tooltip）。

### Q5 此 th switch 的適用範圍

- 題目：`AvlCoursesList` 元件目前用於兩個地方：(1) `CourseStudents` Tab、(2) `Teacher Edit > Learning Tab`（透過 `showToggle=false` 不顯示 switch）。修改範圍要含括哪些？
- 選項：
  - A：**僅修改本 issue 提到的 CourseStudents 表格**，其他用到 `AvlCoursesList` 的地方完全不動
  - B：同時修改所有用到 `AvlCoursesList` 的表格
- **最終決策：A**
- 理由：Issue 明確指向課程學員 TAB，範圍縮到最小避免 side effect。Teacher Edit 頁本來就沒有 switch（`showToggle=false`），不受影響。

### Q6 Teacher 頁是否同步調整

- 題目：是否需要在 Teacher 頁的「學員 TAB」也做同樣改動？
- 選項：
  - A：**只改 Course Edit > 課程學員 TAB**（即 `StudentTable/index.tsx`）
  - B：連 Teacher Edit 相關 TAB 也一起改
- **最終決策：A**
- 理由：Issue 畫面顯示的 URL 是 `/courses/edit/{course_id}`，明確是 Course Edit 頁；Teacher Edit 不在 issue 範圍內。

## 衍生工程決策

基於以上澄清，落地時的工程決策（給 planner / tdd-coordinator 參考）：

1. **`AvlCoursesList` 移除 switch**：拿掉 `showToggle` prop 與內部 switch render；改為純粹接收 `showAllCourses: boolean` prop 由外部控制顯示模式。
2. **`useColumns` 改為 hook 內部管理 `showAllCourses` state**：在 `Granted courses` column 的 `title` 渲染 `欄位標題 + Switch`，column 的 `render` 把 `showAllCourses` 傳給 `AvlCoursesList`。
3. **不影響 Teacher Edit**：Teacher Edit 頁未透過 `useColumns` 使用 `AvlCoursesList`（路徑經 `Teachers/Edit/tabs/Learning`），且原本 `showToggle=false`，仍為「永遠展開全部 avl_courses」行為，與本次調整無關。
4. **i18n**：沿用既有英文 msgid `Show all` / `Show all courses granted to the user`，無需新增字串。
